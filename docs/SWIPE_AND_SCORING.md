# Свайп и оценки команд хакатона: контракт с API

UI **`/jury/swipe`** — карточки команд в стиле dating-app: жест вправо / влево или кнопки задают балл по выбранному критерию.

## Эндпоинты

- **`POST /api/scores/`** — тело: `{ "team_id": number, "criterion_id": number, "value": number }`.  
  Требуется роль **expert** или **admin**. Создаёт или обновляет черновик оценки; `is_final` сбрасывается в `false`.

- **`POST /api/scores/{team_id}/submit`** — помечает **все** оценки текущего эксперта по данной команде как финальные (`is_final: true`), пересчитывает рейтинг и шлёт Socket.IO **`rating_updated`**.

- Лидерборд (`GET /api/scores/leaderboard`) учитывает только строки с **`is_final === true`**.

## Выбранная стратегия (вариант A)

На экране **`/jury/swipe`** жюри выбирает **один активный критерий** из списка `GET /api/criteria/`.

- **Свайп вправо** → `POST /api/scores/` с `value = max_score` выбранного критерия для текущей карточки команды.
- **Свайп влево** → `POST /api/scores/` с `value = 0` (явный «дизлайк» / минимальный балл).

Так один жест всегда привязан к одной паре `(team_id, criterion_id)` при фиксированном выбранном критерии.

## Финализация

После оценки команды по этому критерию жюри нажимает **«Зафиксировать команду»** — вызывается **`POST /api/scores/{team_id}/submit`**. Без этого баллы не попадут в публичный рейтинг.

Для нескольких критериев по одной команде: повторите свайпы (меняя активный критерий в селекте), затем один раз **submit** по `team_id` финализирует все уже введённые оценки эксперта по этой команде.

## Вариант B (не используется в текущем UI)

Стопка карточек «команда × критерий»: каждая карточка несёт свой `criterion_id`; свайп шлёт `POST /api/scores/` с этим критерием. При необходимости можно добавить второй режим позже.
